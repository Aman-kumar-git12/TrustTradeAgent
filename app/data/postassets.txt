Post Assets

Purpose of Post Assets flow
The Post Assets flow allows a seller to publish a new business asset into the TrustTrade marketplace. It is one of the most important seller-side workflows because listing quality directly affects discovery, trust, and lead generation.

Main routes
- /post-asset
- /post-assets/:businessId

Flow structure
The flow happens in two stages:

Stage 1: Business selection
The seller first chooses which business entity the asset should belong to. This is important because TrustTrade ties listings to a business context, not just to a random user account.

Stage 2: Asset creation
After selecting the business, the seller fills in the detailed listing form and submits the asset to the backend.

How it is working
Business selection page
- loads the seller's businesses from the backend
- shows available business cards
- if the seller has no business, it prompts them to create one first
- on selection, it routes the user into the asset posting form for that business

Asset posting page
- loads the target business to show context
- lets the seller enter title
- lets the seller enter description
- supports category selection
- supports custom category input if needed
- collects condition
- collects price
- collects location
- supports images

Image handling
The posting flow supports:
- file upload through the existing image upload endpoint
- direct link-based image addition
- multiple images
- image removal before submission
- limit on how many images can be attached

Submission behavior
When the seller submits the form:
1. the client validates that category and required fields exist
2. the data is sent to the backend asset creation endpoint
3. the business id is included with the asset payload
4. on success, the user is shown a success notification
5. the user is redirected toward the seller dashboard

Why this section is important
This section is where marketplace supply is created. If sellers do not have a clean, guided way to publish assets, Marketplace quality becomes weak. A strong Post Assets flow improves:
- listing completeness
- buyer trust
- search relevance
- seller conversion into leads

What a strong listing should communicate
- what the asset is
- which business is offering it
- condition and category
- price or price signal
- location and logistics implications
- enough description to support buyer confidence

How the AI agent should describe it
The AI agent should explain Post Assets as the seller publishing workflow that starts with selecting a business and ends with submitting a marketplace-ready asset listing that includes commercial, descriptive, and visual information.
